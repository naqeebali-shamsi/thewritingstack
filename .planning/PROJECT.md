# Project: Technical Blog Engine (GSD Edition)

<context>
This project is a precision-engineered writing system designed to produce high-impact technical articles for Mid-Senior Engineers and Tech Leads. It follows the principles of the "Expert Pragmatist" and uses the "Get Shit Done" (GSD) workflow framework.
</context>

<goals>
- Produce MD articles with aggressive hooks and high takeaway density.
- Automate research phases using web tools and MCP.
- Generate image prompts for AI generation.
- Maintain a consistent, authentic technical voice ("I built, I broke").
</goals>

<definitions_of_done>
- Article is in Markdown format.
- Includes context-specific image prompts.
- Contains at least 3 high-value technical takeaways.
- Passes a "Voice Check" (no passive voice, no academic fluff).
- Includes SEO-optimized metadata.
</definitions_of_done>

<tech_stack>
- Structure: GSD 3-layer orchestration.
- Research: Browser Subagent, search_web.
- Formatting: Markdown.
- Generation: Claude 3.5 Sonnet / Claude 3 Opus.
</tech_stack>

<voice_rules>
- **Expert Pragmatist**: Deep technical knowledge + real-world practicality.
- **Authentic & Direct**: Use ownership pronouns (I/We) for internal work.
- **Aggressive Hooks**: No boring intros.
- **Zero Fluff**: If an AI could write it generically, cut it.
</voice_rules>
